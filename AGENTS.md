# AGENTS.md — Cakeculator

## What is this

Cakeculator is a baking calculator app. It helps with recipe scaling, unit conversions, and other baking math. Mobile-first, offline-first SPA/PWA.

## Stack

| Layer | Tech |
|---|---|
| Language | TypeScript (strict mode, `verbatimModuleSyntax`) |
| Runtime | Bun |
| Framework | React 19 |
| Routing | TanStack Router (file-based, auto code-splitting, SPA — no SSR) |
| Bundler | Vite 7 |
| Styling | Tailwind CSS v4 (via `@tailwindcss/vite` plugin) |
| Components | shadcn/ui (new-york style, no RSC) |
| Class utils | CVA + clsx + tailwind-merge |
| Icons | Lucide React |
| Testing | Vitest + Testing Library (React + DOM + jsdom) |
| Linter | OxLint |
| Formatter | OxFmt |

## Design System — Neobrutalism

The visual language is **neobrutalism**: bold borders, high-contrast colors, chunky shadows, flat/solid fills, intentionally "rough" aesthetic.

### Key traits

- **Borders**: Thick, solid, black (`border-border` maps to `oklch(0.14 0 0)` in light mode, `oklch(0.92 0 0)` in dark).
- **Shadows**: Use hard-offset box-shadows (no blur), e.g. `shadow-[4px_4px_0px_0px_var(--border)]`.
- **Colors**: oklch color space throughout. Warm off-white backgrounds, black foreground, yellow/amber accents (`oklch(0.82 0.17 80)`).
- **Corners**: Slightly rounded (`--radius: 0.5rem`) — NOT pill-shaped, NOT sharp 0.
- **Typography**: Manrope (sans — body/UI), Fraunces (serif — headings/display). Both loaded via Google Fonts.
- **Dark mode**: Supported. Toggled via `.dark` class. Inverted palette — dark bg, light borders/text, same accent hue.

### When building UI

- Always use shadcn/ui primitives from `@/components/ui` as the base.
- Apply neobrutalist overrides via Tailwind classes (thick borders, hard shadows, bold weights).
- Use CSS variables from `styles.css` — never hardcode colors.
- Respect the oklch palette. Don't introduce hex/rgb/hsl colors.
- Mobile-first: design for small screens, then scale up.

## Path Aliases

- `#/*` → `./src/*` (Node subpath import, used in package.json `imports`)
- `@/*` → `./src/*` (TypeScript paths, used by shadcn/ui)

Both resolve to `src/`. Use `#/` for application code. `@/` exists for shadcn/ui compatibility.

## Conventions

- **File naming**: PascalCase for components (`Header.tsx`), camelCase for utilities (`utils.ts`), kebab-case for routes (`index.tsx`).
- **Imports**: Use `#/` alias for all `src/` imports.
- **Components**: Functional components only. No class components.
- **Types**: Inline types preferred unless shared. No `as any` or `@ts-ignore`.
- **CSS**: Tailwind utility classes only. No inline styles. No CSS modules.
- **State**: Not yet decided — currently no state management library. Keep state local until a pattern emerges.
- **Data persistence**: Not yet decided — offline-first strategy TBD (likely IndexedDB/localStorage for PWA).
- **Tests**: Co-locate test files next to source or in `__tests__/`. Use Testing Library idioms (query by role/label, not test IDs).

## Auto-generated Files — Do Not Edit

- `src/routeTree.gen.ts` — Generated by TanStack Router plugin.
- `src/components/ui/*` — Managed by shadcn/ui CLI. Customize via Tailwind, not by editing source.
